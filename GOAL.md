# GOAL — Cube 스터디룸 학습 콘텐츠 목표 메타프롬프트

> 이 문서는 모든 학습 콘텐츠(설명·다이어그램·"한눈에"·"큰 그림")를 만들 때의 **기준**이다.
> 작성자(사람·에이전트)는 콘텐츠를 만들기 전에 이걸 먼저 읽고, 만든 뒤 이 기준으로 자가검수한다.

## 미션
Cube가 **이 자료만으로 독학**할 수 있게 한다. 자료가 곧 선생이다. 외부 보충이 필요하면 실패다.

## 학습자 (반드시 이 사람에 맞춘다)
- **ADHD 기질** — 길고 단조로우면 못 버틴다. 짧은 청크 · 시각 먼저 · 동기 먼저.
- **Top-down** — 전체 맥락(지도)을 먼저 잡고 각 부분을 다시 줌인할 때 몰입한다. 모든 레벨에서 "큰 그림 → 디테일".
- **공백 혐오** — 지식 사이에 틈이 있으면 거기서 막힌다. 모든 조각은 앞 지식에 명시적으로 연결돼야 한다.

## 4대 불변 규칙 (하나라도 어기면 그 콘텐츠는 실패)
1. **정확** — 모든 주장은 검증된 사실. 추측·환각·"아마" 금지. API·수식·동작은 실제 소스/공식 문서 근거. 의심되면 확인하고 쓴다.
2. **효율** — 군더더기·중복·뻔한 말 금지. 한 번에 한 개념. 같은 걸 두 번 설명하지 않는다.
3. **몰입** — 다이어그램·비유·동기를 앞세운다. 글벽 금지. 한 문단 3~4문장 이내.
4. **무공백** — `선수 지식 → 지금 → 다음으로`의 사슬을 끊지 않는다. 새 용어는 나온 그 자리에서 한 줄로 정의한다.

## 구조 — 모든 레벨에서 동일한 top-down 줌
1. **스터디룸(홈)**: 주제 지도 → 주제 선택.
2. **주제(대시보드 "큰 그림")**: 전체를 몇 개 스테이지로 묶은 흐름 지도 + "순서가 곧 의존성" 내러티브 → 그다음 파트 로드맵.
3. **파트("한눈에")**: ① 동기 hook("왜 배우나") → ② 핵심 한 줄 → ③ 핵심 흐름 다이어그램 → ④ 선수 지식(앞 파트 링크) ↔ 다음으로 → 그제서야 ⑤ 디테일(개념·코드).
4. **셀**: 코드 → 구어체 해석 → (import면) 라이브러리 미리보기(**이용 순서**) → (알고리즘/구조면) 다이어그램(복잡하면 축약→클릭 펼침) → (핵심 줄이면) hover 풀이.

## 보이스
- 한국어 **구어체**. 컴퓨터/개념에게 지시하듯, 반말 OK. ("이제 노이즈를 더해. 그다음 모델한테 맞히라고 시켜.")
- 다이어그램이 말할 수 있으면 글 대신 다이어그램.
- 용어는 한글(영문 병기), 처음 등장 시 한 줄 정의. 수식은 필요한 만큼만, 유니코드로.

## 만들 때 자문 (체크리스트)
- "한눈에"만 봐도 이 파트의 **왜 / 무엇 / 어디로**가 30초 안에 잡히나?
- 앞 파트를 모르면 막히는 지점이 있나? → 선수 지식에 링크했나?
- 모든 코드 셀에 해석이 1:1로 있나? import는 이용 순서인가? 복잡한 알고리즘에 depth 다이어그램이 있나?
- 틀린 API·수식·주장은 없나? (조금이라도 의심되면 소스 확인 후 작성)
- 지루한 글벽은 없나? 비유·다이어그램·동기로 바꿀 수 있나?

## 수용 기준 (이게 되면 합격)
- 큰 그림만 봐도 전체 흐름이 머릿속에 그려진다.
- 아무 파트나 열어도 "왜 지금 이걸 배우는지"를 즉시 안다.
- 막히면 선수 지식 링크로 스스로 메운다.
- 정확하고, 짧고, 시각적이고, 지루하지 않다.
